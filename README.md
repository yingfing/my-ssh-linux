# my-ssh-linux\n\n[![MIT License](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)\n[![GitHub Actions Build](https://github.com/yingfing/my-ssh-linux/actions/workflows/build.yml/badge.svg)](https://github.com/yingfing/my-ssh-linux/actions/workflows/build.yml)\n\n一个极度精简的、可启动的 Linux 系统，启动后自动开启 SSH 服务，方便你快速获得一个远程命令行环境。\n\n## ✨ 特点\n\n- **体积小巧** – 完整 ISO 约 30 MB\n- **快速启动** – 从 U 盘启动到获得 SSH 服务只需几秒\n- **自动网络** – 通过 DHCP 自动获取 IP 地址（支持有线网卡）\n- **内置 SSH** – 集成 Dropbear 轻量级 SSH 服务器\n- **全自动构建** – 通过 GitHub Actions 一键构建，无需本地编译环境\n- **双启动支持** – 同时支持 BIOS 和 UEFI 启动\n\n## 📦 下载\n\n从 [Releases](https://github.com/yingfing/my-ssh-linux/releases) 页面下载最新的 `custom-system.iso`。\n\n## 🚀 快速开始\n\n### 1. 制作启动盘\n\n- **推荐：Ventoy** – 直接把 ISO 复制到 Ventoy U 盘即可\n- **Linux (dd)** – `sudo dd if=custom-system.iso of=/dev/sdX bs=4M status=progress`\n- **Windows** – 使用 Rufus 以 DD 模式写入\n\n### 2. 启动系统\n\n将 U 盘插入目标电脑，从 U 盘启动。系统启动后会显示：\n\n```text\nStarting DHCP on eth0...\nudhcpc: lease of 192.168.1.100 obtained\n=== SSH system ready. root password is 'root' ===\n/ #\n```\n\n### 3. SSH 登录\n\n在另一台电脑上执行：\n\n```bash\nssh root@<显示的IP地址>\n```\n\n密码：`root`（登录后请立即用 `passwd` 修改）\n\n## 🔧 自定义构建\n\n如果你想修改内核配置、添加驱动或调整启动脚本，可以 Fork 本仓库并触发 GitHub Actions：\n\n1. 进入仓库的 **Actions** 页面\n2. 选择 **Build Minimal SSH System ISO (DHCP)**\n3. 点击 **Run workflow**，填写 Release 标签\n4. 等待构建完成，ISO 会自动发布到 Releases\n\n你可以在 `.github/workflows/build.yml` 中调整：\n- Linux 内核版本\n- BusyBox / Dropbear 版本\n- 内核配置（如添加更多网卡驱动）\n- 启动脚本 (`rcS`)\n\n## 📂 项目结构\n\n```\nmy-ssh-linux/\n├── .github/workflows/\n│   └── build.yml         # GitHub Actions 构建脚本\n├── README.md             # 本文件\n└── LICENSE               # MIT 许可证\n```\n\n> 注意：所有源码和工具都在构建时自动下载，仓库本身不存储二进制文件。\n\n## 📋 技术详情\n\n| 组件          | 版本/说明                          |\n| ------------- | ---------------------------------- |\n| Linux 内核    | 6.6.30（自定义配置）               |\n| 用户态工具    | BusyBox 1.36.1（静态编译）          |\n| SSH 服务      | Dropbear 2024.85                   |\n| 网络配置      | DHCP (udhcpc)                      |\n| 支持的网卡    | Realtek R8169 (有线), RTL8188CE (无线) |\n| 启动方式      | BIOS + UEFI 混合 ISO               |\n\n## ⚠️ 注意事项\n\n- 默认 root 密码为 `root`，**请务必在首次登录后修改**。\n- 无线网卡需要额外配置 WPA（本项目未集成），推荐使用有线网络。\n- 本系统仅用于学习和实验，请勿用于生产环境。\n\n## 🤝 贡献\n\n欢迎提出 Issue 或 Pull Request。如果你希望增加对其他网卡的支持或改进构建流程，请参考 `.github/workflows/build.yml` 中的步骤。\n\n## 📄 许可证\n\n本项目采用 [MIT 许可证](LICENSE)。  \n各组件（Linux 内核、BusyBox、Dropbear 等）均遵循其原始许可证。\n\n## 🙋 常见问题\n\n**Q: 启动后无法获取 IP？**  \nA: 确保网线已插入，且路由器开启了 DHCP。可以手动执行 `udhcpc -i eth0` 重试。\n\n**Q: 如何连接 WiFi？**  \nA: 目前系统未集成 `wpa_supplicant`，建议使用有线网络。如需 WiFi，可在 `rcS` 中添加 `iwconfig` 命令（需要自行编译并集成）。\n\n**Q: 如何保存文件？**  \nA: 本系统运行在内存中（initramfs），重启后所有修改都会丢失。如需持久化，可挂载硬盘或 U 盘分区。\n\n**Q: 构建失败怎么办？**  \nA: 查看 GitHub Actions 的日志，常见原因是网络问题导致源码下载失败。可以重新触发工作流。\n\n---\n\n**Happy Hacking!** 🎉
